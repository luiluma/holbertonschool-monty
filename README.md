<h2>Description:</h2>
<p>- Usage: monty file
        -where file is the path to the file containing Monty byte code
-If the user does not give any file or more than one argument to your program, print the error message 
    -USAGE: monty file, followed by a new line, and exit with the status EXIT_FAILURE
-If, for any reason, it’s not possible to open the file, print the error message Error: Can't open file <file>, followed by a new line, and exit with the status EXIT_FAILURE
    -where <file> is the name of the file
-If the file contains an invalid instruction, print the error message L<line_number>: unknown instruction
<opcode>, followed by a new line, and exit with the status EXIT_FAILURE
    -where is the line number where the instruction appears.
    -Line numbers always start at 1
-The monty program runs the bytecodes line by line and stop if either:
    -it executed properly every line of the file
    -it finds an error in the file
    -an error occured
-If you can’t malloc anymore, print the error message Error: malloc failed, followed by a new line, and exit with status EXIT_FAILURE.
-You have to use malloc and free and are not allowed to use any other function from man malloc (realloc, calloc, …)</p>

<h2> LIFO / FIFO </h2>
<p>FIFO is an abbreviation for first in, first out. It is a method of handling data structures where the first element is processed first and the newest element is processed last. </p>
<p>LIFO is an abbreviation for Last In, First Out is the same as First In, Last Out (FILO). It is a method for handling data structures where the last element is processed first and the first element is processed last. </p>
